## ✨ تغییرات نسخه v1.3 (Core Hardening)
این نسخه ترکیبی از پایداری نسخه‌های قبل همراه با سخت‌گیری‌های امنیتی جدید است:

- ⏱️ **مدیریت پیشرفته Timeout:** اضافه شدن `UPSTREAM_TIMEOUT_MS` برای کنترل اتصال. پیش‌فرض روی `120000` تنظیم شده است.
- 🛡️ **محدودیت متدها:** جهت افزایش امنیت، تنها متدهای `GET`, `HEAD` و `POST` مجاز هستند.
- 🧹 **فیلترینگ هوشمند هدرها:** حذف دقیق‌تر هدرهای اضافی و سخت‌گیری بیشتر روی هدرهای *hop-by-hop/proxy*.
- 🔑 **احراز هویت ایمن:** مسیر auth از Query به طور کامل حذف شد. دسترسی فقط از طریق هدر `x-relay-key` کنترل می‌شود.
- 🛣️ **مسیریابی سخت‌گیرانه:** متغیر `RELAY_PATH` اکنون اجباری است (Fail-Closed با ارور `500` در صورت عدم تنظیم).
- 🐛 **دیباگینگ:** بهبود سیستم لاگ خطا و تایم‌اوت برای عیب‌یابی سریع‌تر.

## ⚙️ راهنمای استقرار دستی روی Vercel (CLI)

برای امنیت بیشتر، پروژه را روی سیستم خود دانلود کرده و به صورت دستی آپلود می‌کنیم.

### ۱. آماده‌سازی فایل‌ها
ابتدا این ریپازیتوری را به صورت فایل **ZIP** دانلود کرده (از دکمه سبز رنگ Code -> Download ZIP) و در سیستم خود Extract (استخراج) کنید.

### ۲. نصب ابزار Vercel و ورود به اکانت
در سیستم خود ترمینال (یا Git Bash) را باز کنید و ابزار ورسل را به صورت گلوبال نصب کنید (نیاز به Node.js دارید):
```bash
npm i -g vercel
```
پس از نصب، وارد مسیر پوشه اکسترکت شده بشوید:
```bash
cd /path/to/your/extracted/folder
```
سپس به اکانت ورسل خود لاگین کنید (یک پنجره مرورگر برای تایید باز می‌شود):
```bash
vercel login
```

### ۳. استقرار پروژه (Deploy)
حالا در همان پوشه، دستور زیر را برای شروع فرآیند دیپلوی وارد کنید:
```bash
vercel deploy
```
ورسل از شما چند سوال می‌پرسد که باید به این شکل پاسخ دهید:
- **Set up and deploy?** 👉 (`Y` را تایپ کنید و Enter بزنید)
- **Which scope do you want to deploy to?** 👉 (اکانت خود را با فلش کیبورد انتخاب کنید)
- **Link to existing project?** 👉 (`N` را تایپ کنید)
- **What's your project's name?** 👉 (یک نام دلخواه برای پروژه‌تان بنویسید)
- **In which directory is your code located?** 👉 (فقط `Enter` بزنید)
- **Auto-detected Project Settings:** 👉 (اگر تنظیمات پیش‌فرض را نشان داد، با `Enter` تایید کنید)

> منتظر بمانید تا کدهای شما روی سرورهای ورسل آپلود شده و لینک `Production` به شما داده شود.

### ۴. تنظیم متغیرهای محیطی (Environment Variables)
پس از پایان دیپلوی موفق:
1. وارد سایت Vercel شده و روی پروژه جدید خود کلیک کنید.
2. به مسیر `Settings -> Environment Variables` بروید و مقادیر زیر را اضافه کنید:

| متغیر (ENV) | وضعیت | کاربرد | مثال / توضیحات |
| :--- | :---: | :--- | :--- |
| `TARGET_DOMAIN` | 🔴 **اجباری** | آدرس سرور اصلی (Upstream) | `https://1.2.3.4:8443` |
| `RELAY_PATH` | 🔴 **اجباری** | مسیر مجاز کانکشن | `/api` *(حتماً با `/` شروع شود)* |
| `RELAY_KEY` | ⚪ اختیاری | قفل دسترسی (احراز هویت) | حداقل ۱۶ کاراکتر |
| `UPSTREAM_TIMEOUT_MS`| ⚪ اختیاری | زمان تایم‌اوت (میلی‌ثانیه) | `120000` (حالت متعادل و پیش‌فرض) |

### ۵. استقرار مجدد (Redeploy)
برای اعمال شدن متغیرهایی که در مرحله قبل ثبت کردید، در همان سایت ورسل از منوی سمت چپ به تب **Deployments** بروید، روی دیپلوی اخیر خود کلیک کرده و گزینه **Redeploy** را بزنید.

## 💻 تنظیمات سمت کلاینت (نمونه کانفیگ)

برای اتصال به این Relay با کلاینت‌های استاندارد Xray/V2ray، از فرمت زیر استفاده کنید:
```text
vless://UUID-HERE@react.dev:443?encryption=none&security=tls&sni=react.dev&fp=chrome&alpn=h2&insecure=0&allowInsecure=0&type=xhttp&host=YOUR-DOMAIN-FROM-VERCEL&path=%2Fapi&mode=auto#Vercel
```
**نکات مهم کانفیگ بالا:**
- به جای `YOUR-DOMAIN-FROM-VERCEL` دامنه پروژه‌ای که ورسل به شما داده است را قرار دهید.
- عبارت `%2Fapi` در بخش `path` معادل مسیر `/api` است. اگر متغیر `RELAY_PATH` خود را روی چیز دیگری تنظیم کرده‌اید، آن را آپدیت کنید (کاراکتر `%2F` نشان‌دهنده اسلش است).

## 🛠️ کدهای وضعیت سیستم (Status Codes)
در صورت بروز مشکل، سیستم یکی از کدهای استاندارد زیر را برمی‌گرداند:

- 🟢 `200` : اتصال موفق
- 🟡 `403` : کلید دسترسی (`RELAY_KEY`) اشتباه است.
- 🟠 `404` : مسیر درخواستی نامعتبر است (با `RELAY_PATH` تطابق ندارد).
- 🔴 `405` : متد درخواستی غیرمجاز است.
- 🔴 `500` : خطای داخلی یا ناقص بودن متغیرهای محیطی (ENV).
- 🟣 `504` : تایم‌اوت شدن اتصال به سرور Upstream.
