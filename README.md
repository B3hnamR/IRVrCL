# VrcLIraniCore

این پروژه یه Relay سبک برای XHTTP روی Vercel Edge Runtime هست.
هدفش اینه که هم سرعت خوب بده، هم مصرف بی‌خودی کمتر بشه، هم ریسک abuse پایین بیاد.

---

## Changelog v1.3

این نسخه در واقع ترکیب کامل تغییرات 1.2 + سخت‌گیری‌های جدیده:

- کنترل timeout برای اتصال به upstream اضافه شد با ENV زیر:
  - `UPSTREAM_TIMEOUT_MS`
- مقدار پیش‌فرض timeout از حالت قبلی بهتر شد و الان برای استریم‌های طولانی‌تر روی:
  - `120000`

- محدودیت متدها اعمال شد:
  - فقط `GET`, `HEAD`, `POST`

- فیلتر هدرها بهینه‌تر شد:
  - هدرهای اضافی و غیرضروری کمتر forward می‌شن
  - لیست strip هم سخت‌گیرانه‌تر شده برای هدرهای hop-by-hop/proxy

- لاگ خطا/timeout بهتر شد تا عیب‌یابی راحت‌تر باشه

- `RELAY_PATH` دیگه شل و ول نیست:
  - باید حتما از ENV ست بشه
  - اگر ست نشه یا `/` باشه، سرویس fail-closed میده (`500`)

- `RELAY_KEY` اختیاریه:
  - اگر ست کنی، فقط از هدر `x-relay-key` قبول میشه
  - اگر ست شده باشه و کمتر از 16 کاراکتر باشه، سرویس `500` میده
  - اگر ست شده باشه و کلید اشتباه باشه، درخواست `403` میگیره

- مسیر auth از query کامل حذف شده:
  - یعنی دیگه `k` توی query قابل قبول نیست

- rewrite پروژه روی حالت عمومیه:
  - `/(.*) -> /api/index`
  - ولی مسیر واقعی مجاز رو خود کد با `RELAY_PATH` چک می‌کنه

## Deploy روی Vercel

### 1) پروژه رو ایمپورت کن

- ریپو رو روی Vercel import کن
- برنچ: `main`

### 2) Environment Variables رو ست کن

از این مسیر:
`Project -> Settings -> Environment Variables`

این 4 تا رو بزن:

1. `TARGET_DOMAIN` (Required)
- کارش: آدرس سرور اصلی upstream
- فرمت: `https://host:port`
- مثال:
  - `https://your-origin.example.com:2053`
  - `https://1.2.3.4:8443`

2. `RELAY_PATH` (Required)
- کارش: مسیر مجاز relay
- باید با `/` شروع بشه
- نباید فقط `/` باشه
- مثال:
  - `/vc`
  - `/edge-vc-901`
  - `/r/ab92`

3. `RELAY_KEY` (Optional)
- کارش: قفل دسترسی relay
- اگر ستش کنی، باید از سمت کلاینت توی هدر فرستاده بشه:
  - `x-relay-key: <RELAY_KEY>`
- حداقل طول: 16 کاراکتر
- پیشنهاد واقعی: 32 تا 64 کاراکتر

4. `UPSTREAM_TIMEOUT_MS` (Optional)
- کارش: زمان timeout برای upstream
- پیش‌فرض اگر نزنی: `120000`
- پیشنهاد:
  - `90000` اگر میخوای زودتر timeout بشه
  - `120000` حالت متعادل
  - `180000` اگر استریم طولانی‌تر داری

### 3) Redeploy یادت نره

بعد از هر تغییر ENV حتما Redeploy بزن، وگرنه تنظیم جدید اعمال نمیشه.

## تنظیم سمت کلاینت

- `Path` باید دقیقا برابر `RELAY_PATH` باشه
- `Host` باید دامنه پروژه Vercel (یا دامنه شخصی خودت) باشه
- اگر `RELAY_KEY` ست شده، هدر زیر باید ارسال بشه:
  - `x-relay-key: <RELAY_KEY>`

## کدهای وضعیت رایج

- مسیر نامعتبر: `404`
- متد نامعتبر: `405`
- ENV ناقص یا اشتباه: `500`
- کلید اشتباه: `403`
- timeout upstream: `504`
