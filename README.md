## ✨ تغییرات نسخه v1.3 (Core Hardening)
این نسخه ترکیبی از پایداری نسخه‌های قبل همراه با سخت‌گیری‌های امنیتی جدید است:

- ⏱️ **مدیریت پیشرفته Timeout:** اضافه شدن `UPSTREAM_TIMEOUT_MS` برای کنترل اتصال. پیش‌فرض روی `120000` (مناسب برای استریم‌های طولانی) تنظیم شده است.
- 🛡️ **محدودیت متدها:** جهت افزایش امنیت، تنها متدهای `GET`, `HEAD` و `POST` مجاز هستند.
- 🧹 **فیلترینگ هوشمند هدرها:** حذف دقیق‌تر هدرهای اضافی و سخت‌گیری بیشتر روی هدرهای *hop-by-hop/proxy*.
- 🔑 **احراز هویت ایمن:** مسیر auth از Query به طور کامل حذف شد (`k` دیگر قابل قبول نیست). دسترسی فقط از طریق هدر `x-relay-key` کنترل می‌شود.
- 🛣️ **مسیریابی سخت‌گیرانه:** متغیر `RELAY_PATH` اکنون اجباری است (Fail-Closed با ارور `500` در صورت عدم تنظیم).
- 🐛 **دیباگینگ:** بهبود سیستم لاگ خطا و تایم‌اوت برای عیب‌یابی سریع‌تر.

## ⚙️ راهنمای استقرار (Deploy) روی Vercel

### ۱. ایمپورت پروژه
ابتدا ریپازیتوری را در حساب Vercel خود ایمپورت کنید (برنچ `main`).

### ۲. تنظیم متغیرهای محیطی (Environment Variables)
به مسیر `Project -> Settings -> Environment Variables` بروید و مقادیر زیر را تنظیم کنید:

| متغیر (ENV) | وضعیت | کاربرد | مثال / توضیحات |
| :--- | :---: | :--- | :--- |
| `TARGET_DOMAIN` | 🔴 **اجباری** | آدرس سرور اصلی (Upstream) | `https://1.2.3.4:8443` |
| `RELAY_PATH` | 🔴 **اجباری** | مسیر مجاز کانکشن | `/edge-vc-901` *(حتماً با `/` شروع شود)* |
| `RELAY_KEY` | ⚪ اختیاری | قفل دسترسی (احراز هویت) | حداقل ۱۶ کاراکتر (پیشنهاد: ۳۲ تا ۶۴) |
| `UPSTREAM_TIMEOUT_MS`| ⚪ اختیاری | زمان تایم‌اوت (میلی‌ثانیه) | `120000` (حالت متعادل و پیش‌فرض) |

### ۳. استقرار مجدد (Redeploy)
> **⚠️ بسیار مهم:** بعد از هر بار تغییر در Environment Variables، حتماً پروژه را **Redeploy** کنید تا تنظیمات جدید روی Edge Network اعمال شوند.

## 💻 تنظیمات سمت کلاینت

برای اتصال صحیح کلاینت به این Relay، مقادیر زیر باید دقیقاً ست شوند:

- **Path:** باید دقیقاً برابر با مقدار `RELAY_PATH` باشد.
- **Host / SNI:** باید دامنه پروژه Vercel (یا دامنه شخصی متصل به آن) باشد.
- **Header:** در صورت تنظیم `RELAY_KEY`، ارسال هدر زیر الزامی است:
  ```http
  x-relay-key: <YOUR_RELAY_KEY>
  ```

## 🛠️ کدهای وضعیت سیستم (Status Codes)
در صورت بروز مشکل، سیستم یکی از کدهای استاندارد زیر را برمی‌گرداند:

- 🟢 `200` : اتصال موفق
- 🟡 `403` : کلید دسترسی (`RELAY_KEY`) اشتباه است.
- 🟠 `404` : مسیر درخواستی نامعتبر است (با `RELAY_PATH` تطابق ندارد).
- 🔴 `405` : متد درخواستی غیرمجاز است.
- 🔴 `500` : خطای داخلی یا ناقص بودن متغیرهای محیطی (ENV).
- 🟣 `504` : تایم‌اوت شدن اتصال به سرور Upstream.
